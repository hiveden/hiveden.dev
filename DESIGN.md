# DESIGN.md — 设计语言 M

## 定位

设计语言 **M** 强调内容优先、极简主义与技术极客氛围，提供无干扰的沉浸式阅读体验。主要用于个人品牌站点及配套项目。

## 设计理念

**"每个像素都承载信息，克制即美。"**

### 原则
1. **信息优先** — 减少视觉噪音，专注内容呈现。
2. **克制** — 单一强调色，深色模式优先。
3. **一致** — 与所有配套内容生态（包括 Astral Video 视频生成项目）共享底层的视觉语言习惯。

## 颜色体系

基于深色背景（Dark Mode First），统一使用 CSS Variables 在全局定义。

| Token | 值 | CSS Variable | 用途 |
|-------|-----|--------------|------|
| `background` | `#09090B` | `--color-background` | 页面全局纯深背景 |
| `foreground` | `#FAFAFA` | `--color-foreground` | 核心文字内容、主标题 |
| `accent` | `#A855F7` | `--color-accent` | 唯一的强调色 (紫色)、品牌色、链接交互 |
| `muted` | `#A1A1AA` | `--color-muted` | 次要文字、描述、副标题、引用文本 |
| `subtle` | `#71717A` | `--color-subtle` | 极弱的辅助信息、标签、无修饰分隔 |
| `surface` | `#18181B` | `--color-surface` | 卡片背景、代码块底色、表格头部 |
| `border` | `#27272A` | `--color-border` | 分隔线、代码块边框、内容区块划分 |

## 字体体系

| 用途 | 字体 | CSS Variable | 场景 |
|------|------|--------------|------|
| 正文/标题 | Geist | `--font-sans` | 绝大部分阅读内容、标题层级 |
| 代码/数据 | Geist Mono | `--font-mono` | 内联代码、代码块、数据指标、终端效果 |

## 特殊视觉效果

作为技术站，保留部分极客元素的视觉语言：

### 1. 终端打字光标 (Blink Cursor)
用于模拟命令行输入效果。
- **类名**：`.animate-blink`
- **动画**：步进式闪烁 (step-end)

### 2. 网格背景 (Grid Background)
强调极客与工程感，通常作为 Hero 区或大面积留白区域的底纹。
- **类名**：`.bg-grid`
- **样式**：基于 `accent` 的 3% 透明度的线性渐变网格。

### 3. 微发光 (Glow Accent)
局部点缀，通常响应鼠标交互或特定区域的高光。
- **类名**：`.glow-accent`
- **样式**：基于 `accent` 的弱径向渐变 (`rgba(168, 85, 247, 0.08)`)。

## Prose 文档排版规范

对于 Markdown 渲染或大段长文本阅读，需遵守以下全局排版：
- **行高**：宽松的 `1.8`，字号 `0.9375rem`。
- **链接**：必须有交互响应。下划线采用 `accent` 和透明度混色的渐变过渡。
- **引用 (Blockquote)**：左侧加入 `accent` 3px 边框，背景使用混色的 `surface`。
- **代码 (Code)**：内联代码带 `border` 和 `surface` 背景，长代码块使用极深背景 `#0d1117` 以区分主阅读区。
